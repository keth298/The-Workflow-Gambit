ENGINE FEATURES
───────────────
Board:
  - 64-square mailbox representation
  - Incremental Zobrist hashing (XOR-based)
  - Push/pop move history (no full-copy; O(1) make/unmake)
  - FEN loader (all standard fields)

Move Generation:
  - Pseudo-legal + legality filter (is-king-in-check test)
  - Pawns: single/double push, diagonal capture, en passant, promotion (Q/R/B/N)
  - Knights, bishops, rooks, queens: standard delta/ray generation
  - Kings: 8 adjacent squares + castling (KQkq)
  - Castling: path-attack check (passes through/from attacked square blocked)

Search:
  - Iterative deepening negamax
  - Alpha-beta pruning (fail-hard)
  - Principal variation tracked via transposition table
  - Transposition table (dict, unbounded, depth-preferred replacement)
  - Quiescence search (captures + promotions only)
  - Null-move pruning (R=3, disabled in check and pawnless positions)
  - Late-move reduction (LMR, depth>=3, move>=4, -1 ply reduction)
  - Move ordering: TT move > captures (MVV-LVA) > promotions > killers > history heuristic
  - Two killer slots per ply
  - History heuristic (depth² bonus on quiet cutoff moves)

Evaluation:
  - Material values: P=100, N=320, B=330, R=500, Q=900
  - Piece-square tables (PSTs) for all 6 piece types
  - PSTs are mirrored for black (vertical flip)
  - Score returned relative to side to move (negamax convention)

Time Management:
  - Fraction-of-clock: target = remaining_time/30 + increment/2
  - Early-exit if >50% of target used between depths
  - Hard stop if >90% of target used
  - movetime= supported directly
  - go infinite supported (1e9 second limit, stop via 'stop' command)
  - Threaded search (go runs in daemon thread, main loop stays responsive)

UCI Compliance:
  - uci → id name/author + uciok
  - isready → readyok
  - ucinewgame → reload startpos, clear TT
  - position startpos [moves ...]
  - position fen <fen> [moves ...]
  - go [wtime/btime/winc/binc/movetime/depth/infinite]
  - stop → sets stop flag, joins thread
  - quit → graceful exit
  - All output on stdout, flushed immediately
  - All logs (none by default) would go to stderr
